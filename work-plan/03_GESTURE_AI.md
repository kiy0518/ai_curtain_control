# 03. 제스처 / AI 견고화

> Phase 1 · 목표버전 v0.2.0 · 상태: 진행중(기본 제스처 동작)

## 현재 상태
- YOLOv8-pose 손키포인트(21점) NPU 추론 동작.
- 제스처: 열림(✋ 4손가락)/닫힘(✊ 0~1)/정지(✌️ 검지+중지). 떨림방지 5프레임.
- ⚠️ **정지(브이) 인식 불안정** — 모델의 접힌 손가락 키포인트가 부정확.
- ✅ **선택형 모델 프로파일 도입**(`src/profiles.py`) — 모델 교체 시 키포인트/제스처/스켈레톤이 함께 전환.

## 모델 프로파일 (선택형) — 거리별 대응
원거리에선 손이 작아 키포인트가 안 잡힘 → **전신 포즈(17점)** 로 팔 동작 인식이 거리에 강함.
`serve.py --profile <name>` 으로 선택(대시보드 모델선택과 연동 예정).

| 프로파일 | 모델 | 키포인트 | 거리 | 제스처 |
|----------|------|----------|------|--------|
| `hand_near` | YOLOv8-pose hand | 21 | 근거리 | 손바닥=열림 / 주먹=닫힘 / 브이=정지 |
| `body_far` | YOLOv8-pose body(COCO) | 17 | **원거리** | 양팔위=열림 / 가슴앞교차=닫힘 / 한팔수평(T)=정지 |

- 전신 제스처 판정은 **어깨폭 기준 상대값**이라 거리/스케일 무관. (`src/gesture_body.py`)
- 두 모델 다 보드 `models/` 에 두고 런타임 전환. 합성 테스트 통과(양 디코드·양 제스처).
- 모델 변환: `convert/README.md` "전신 포즈" 절 (우리 검증된 경로 동일, `yolov8n-pose.pt`, imgsz=640).

## 목표
- 3제스처 안정 인식 + 오작동 방지 + 부분개폐 + (선택)사용자 식별.

## 작업
### 3.1 정지 제스처 안정화
- [ ] `--debug` 로 주먹/브이 시 `n`(펴진 손가락) 값 수집·분석
- [ ] 펴짐 판정식 개선: 손목-MCP 기준, 또는 PIP 관절 각도(직선도) 기반
- [ ] 키포인트 신뢰도(conf) 낮은 손가락은 판정 제외
- [ ] 필요시 입력크기 상향(320/640)로 키포인트 정확도 확보

### 3.2 오작동 방지
- [ ] 활성화 제스처(예: 손 들고 N초 유지) 후에만 명령 수신
- [ ] 제스처 확정 후 쿨다운(중복 트리거 방지)
- [ ] 손 1개만/특정 거리 범위에서만 인식

### 3.3 부분 개폐
- [ ] 손가락 개수 → 개폐량 매핑(1→25%,2→50%,3→75%,5→100%) 또는 손높이 기반
- [ ] control 상태머신과 연동(`02_*`)

### 3.4 모델 품질
- [ ] 현 사전학습 모델 한계(50% 학습) 평가
- [ ] 대안: hand-keypoints로 YOLO11n-pose 재학습(Colab GPU) 또는 더 나은 가중치
- [ ] INT8 양자화 비교(속도 vs 정확도), 입력크기 최종 선택

### 3.5 (선택) 사용자 식별 → 개인 맞춤
- [ ] 얼굴 인식(별도 경량 모델) 또는 등록 제스처/태그
- [ ] 식별된 사용자 → 개인 프로파일(선호 개폐량/스케줄) 자동 적용 (`05_*`)

## 오픈 이슈
- 사용자 식별은 프라이버시 고려(로컬 처리, 옵트인).
- NPU 동시 멀티모델(손+얼굴) 부하 검토.
