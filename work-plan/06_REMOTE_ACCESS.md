# 06. 원격 접속 · 고유 주소 생성

> Phase 5 · 목표버전 v0.6.0 · 상태: 미착수
> 참고: `kiy0518/Azone_Gateway` (디바이스 등록기 + 게이트웨이 패턴)

## 목표
집 밖에서도 대시보드 접속/제어. 각 디바이스에 **고유 주소**를 발급해 NAT/방화벽 뒤에서도 도달 가능.

## 접근 방식 (택1, 결정 필요)
| 방식 | 설명 | 장단점 |
|------|------|--------|
| A. 자체 게이트웨이(등록기) | 디바이스가 중앙 서버에 등록 → `https://<id>.도메인` 로 라우팅 (Azone 패턴) | 완전한 통제·브랜딩 / 서버·도메인 운영 필요 |
| B. cloudflared 터널 | Cloudflare Tunnel로 보드↔클라우드 상시 연결, 고유 서브도메인 | 무료/간단·HTTPS 자동 / Cloudflare 의존 |
| C. FRP | 자체 VPS에 frps, 보드에 frpc → 고유 포트/서브도메인 | 가볍고 통제 가능 / VPS 필요 |

> 1차 권장: **B(cloudflared)** 로 빠르게 원격 확인 → 제품화 시 **A(자체 등록기)** 로 고유주소·다중기기 관리.

## 자체 등록기(A) 설계 개요
- 디바이스 부팅 시 **등록기**가 중앙 서버에 device_id/공개키로 등록 → 고유 주소/토큰 수령.
- 상시 터널(WebSocket/역방향 연결) 유지, 중앙 서버가 외부 요청을 해당 디바이스로 프록시.
- `deploy/`에 systemd 서비스(부팅 시 자동 등록/연결).
