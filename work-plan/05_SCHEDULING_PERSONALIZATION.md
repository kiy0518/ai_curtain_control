# 05. 스케줄 · 자동화 · 개인 맞춤

> Phase 4 · 목표버전 v0.5.0 · 상태: 미착수

## 목표
손짓 없이도 "알아서" 여닫는 자동화 + 사용자별 맞춤.

## 기능
### 스케줄
- [ ] 시간 기반(예: 07:00 열기, 22:00 닫기), 요일별
- [ ] 일출/일몰 연동(`astral`, 위치 좌표 기반) — "일출 30분 후 열기"
- [ ] 일회성/반복 예약

### 자동화(센서/비전)
- [ ] 조도 기반: 카메라 밝기 또는 조도센서 → 밝으면 닫기/어두우면 열기(옵션)
- [ ] 재실 감지: 사람 입실 시 열기(사람 검출 모델/PIR)
- [ ] 외출/취침 모드

### 개인 맞춤
- [ ] 사용자 프로파일(선호 개폐량/스케줄)
- [ ] (선택) 사용자 식별(`03.5`)과 연동해 자동 적용
- [ ] 사용 이력 기반 제안("보통 이 시간에 닫으시네요")

## 설계
- `scheduler/`: APScheduler(cron/interval) + 이벤트 버스 → control 명령.
- 설정/프로파일/이력: SQLite. 대시보드(`04_*`)에서 CRUD.
- 우선순위 규칙: 수동(제스처/버튼) > 자동, 충돌 해결 정책 정의.

## 작업
- [ ] 스케줄 데이터 모델(SQLite) + CRUD API
- [ ] APScheduler 통합 → control 호출
- [ ] astral 일출/일몰 계산(좌표 설정)
- [ ] 조도/재실 자동화(옵션, 센서 결정 후)
- [ ] 프로파일·이력 저장

## 오픈 이슈
- 위치(좌표)·시간대 설정 입력 방법.
- 자동 vs 수동 충돌 정책(자동이 수동을 덮어쓰지 않도록).
