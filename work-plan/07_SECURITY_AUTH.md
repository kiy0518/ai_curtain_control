# 07. 인증 · 보안

> Phase 5 · 목표버전 v0.6.0 · 상태: 미착수

## 목표
대시보드/제어 API를 인증된 사용자만 접근. 원격 노출 전 필수.

## 작업
- [ ] 대시보드 로그인(사용자/비밀번호, 세션/JWT)
- [ ] 역할 분리: 사용자 vs 관리자(모델/시스템 설정은 관리자만)
- [ ] API 토큰(외부/모바일 클라이언트용)
- [ ] 디바이스 인증(원격 게이트웨이 등록 시 키 기반)
- [ ] HTTPS(로컬: 자체서명/내부, 원격: 터널이 TLS 종단)
- [ ] 비밀값 관리(.env, 토큰 평문 커밋 금지) + 레이트리밋/로그인 시도 제한

## 설계 메모
- 로컬 단독 사용 단계에선 간단한 단일 비밀번호로 시작 → 원격 활성화 시 강화.
- 제어 명령(모터)은 인증 + (선택)2차 확인으로 오작동/악용 방지.

## 오픈 이슈
- 사용자 수/멀티 계정 필요 여부.
- 토큰/비밀번호 저장소(SQLite + 해시).
