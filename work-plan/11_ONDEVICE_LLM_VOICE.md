# 11. 온디바이스 AI — 데이터 분석(맞춤형 학습) + 음성 인식

> Phase 8 · 목표버전 v1.2.0 · 상태: 미착수
> 결정(사용자): **클라우드 미사용**(사용료 발생 비추) → **온디바이스 전용**.

두 가지 온디바이스 AI 기능:
1. **축적 데이터 분석 → 개인 맞춤** : 사용 이력(제스처/스케줄/개폐 시각)을 분석해 사용자 패턴을 학습하고 자동 스케줄/추천을 생성.
2. **음성 인식(STT)** : "커튼 닫아" 같은 음성 명령을 보드에서 직접 인식(오프라인).

```
[마이크] → 웨이크워드 → STT(오프라인) → 의도분류 → CurtainController
[이벤트 로그(SQLite)] → 주기적 분석(LLM/규칙) → 맞춤 스케줄/추천
```

## 모델/런타임 선택 (RK3576, 4GB 제약 고려)
- ⚠️ 이 보드는 **PyTorch 불가(SIGILL)** → **GGUF(llama.cpp) / RKLLM(NPU) / ONNX** 만.
- ⚠️ **메모리 4GB + NPU를 비전과 공유** → 대형 모델 불가. **소형 + 온디맨드** 원칙.

### 음성 인식(STT)
- [ ] 후보: **vosk(한국어 소형)** 또는 **sherpa-onnx**(오프라인, 경량) / **whisper.cpp tiny~base**(한국어 가능, 무거움)
- [ ] **웨이크워드**(openWakeWord / Porcupine 등)로 상시 청취 → 호출 시에만 STT 구동(자원 절약)
- [ ] **마이크 하드웨어** 필요(USB 마이크 또는 I2S) — 보드 입력장치 확인/추가
- [ ] 인식 텍스트 → 의도분류(간단 키워드 규칙 또는 소형 LLM) → 명령

### 데이터 분석 / 맞춤 학습
- [ ] 이벤트 로깅 강화: 개폐·제스처·스케줄 실행·수동조작을 SQLite `events`에 적재(시각/소스/동작)
- [ ] 분석 방식 2안:
  - (가벼움) **규칙/통계**: 시간대별 개폐 빈도 → 자동 스케줄 제안(LLM 불필요, 저자원)
  - (고급) **소형 LLM(RKLLM)**: Gemma 3n E2B / Qwen2.5‑1.5B 로 패턴 요약·자연어 추천 — **온디맨드**(분석 시에만 로드)
- [ ] 추천 → 대시보드에 "이 시간대에 자주 닫으시네요 — 스케줄 추가?" 형태로 노출(`04_*`/`05_*` 연계)

## 모델 비고 (참고)
- **Gemma 4**(2026‑04)는 RKLLM 미지원(현재 Gemma 2/3/3n까지). 보드 온디바이스는 **Gemma 3n E2B / Qwen2.5‑1.5B** 권장.
- RKLLM 사전변환 모델 제공: Qwen2.5‑1.5B, Llama‑3.2‑1B, Gemma‑2B 등.

## 오픈 이슈 / 결정 필요
- 마이크 종류(USB vs I2S), 한국어 STT 정확도/지연 목표.
- 분석은 규칙/통계로 충분한지 vs LLM 필요 수준.
- 비전+STT(+LLM) 동시 구동 시 메모리/NPU 경합 → 온디맨드/우선순위 정책.
- 4GB로 부족하면 더 큰 RAM 보드(RK3588 8GB 등)로의 확장 고려.
